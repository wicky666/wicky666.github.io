Aura组件的生命周期

什么是组件的生命周期？

生命周期，故名思意，就是一个物体，从出生，成长，再到它毁灭所经历的一系列阶段。组件也有生命周期的概念，用来描绘组件从创建到销毁的一个过程,在这过程中,每个特定的阶段都会触发一些方法。通常呢，一个组件完整的生命周期中都会经历下面的过程:

  • 构造 → 组件实例化后的准备阶段
  • 渲染 → 组件被添加到dom tree，接下去要进行的渲染阶段
  • 更新 → 用户操作组件，组件的更新阶段
  • 移除 → 组件用完后从dom tree中移除

DOM(Document Object Modal)

通过Lightning框架提供的custom renderer函数可以管理和更新组件预定义的DOM结构。

那什么是DOM呢？

HTML和DOM就像是两种不同语言的相同文档，而DOM则被表示为一个JavaScript对象。如果我们修改DOM上的任何东西，HTML也会自动刷新。如果我们在HTML上添加新的元素，DOM也会随之改变，这都是同步发生的，浏览器会去处理这些事。

Lightning组件的生命周期

Lightning Aura组件在其整个生命周期中也会经历构建、渲染，更新，移除的过程。
和Lightning Web Component组件不一样，Aura组件的生命周期函数在实际的开发阶段用到的地方不会很多，因为组件的初始化,渲染的整个过程都是由lightning框架完成的。

我们只需定义Init事件来准备组件初期加载所需要的数据就OK。

大多数组件都不需要在初始化时更改DOM结构，所以大多数时候只要记住官方提供的几个生命周期内置函数会在哪些时点被触发就可以了。
但是，如果我们需要在组件初始化时改变DOM结构, 或者想要在组件插入DOM之后与生成的DOM树交互, 可以考虑以下方式：

  1. 开箱即用的render事件
  2. renderer.js创建自定义的渲染逻辑。

开箱即用的render事件

当组件被渲染(rendered)或重新渲染(rerendered)时，render event会被触发。该事件可以对DOM进行后处理 或者 对组件的渲染(rendered)或重新渲染(rerendered)作出响应，与在renderer.js中重写afterRender()方法相比，render event更容易使用。

render event事件类似于组件初始化用到的init事件处理程序，他们都是在组件的标记中调用。

用法:
在controller.js中中执行onRender Action处理函数，action属性中可以定义任意名称的方法。

创建Custom Renderer.jsr（自定义渲染器）

以下是Salesforce官方对Custom Renderer的描述：

Custom Renderer
You don’t normally have to write a custom renderer, but it’s useful when you want to interact with the DOM tree after the framework’s rendering service has inserted DOM elements. If you want to customize rendering b>ehavior and you can’t do it in markup or by using the init event, you can create a client-side renderer.

如上所述，在大多数情况下，我们不需要去添加Custom Renderer来改变组件默认的渲染行为。

只有在以下情况才需要去添加自定义渲染逻辑:

  1. 组件渲染完毕后，需要与DOM树交互。
  2. 组件加载时，通过cmp中的标签或init事件无法实现的渲染行为。

在组件Bundle中定义RENDERER.js

组件在Render(渲染), Rerenderr(重新渲染)或Unrender(被销毁)时，会自动调用不同的渲染函数。我们可以重写这些函数来改变组件的默认渲染行为。

让我们看一下这些函数在组件加载过程中的执行顺序。

  1. Init event – 在组件实例化之后被触发，因为发生在构建阶段，通常在此阶段为组件准备初期数据。
  2. render() – 组件渲染阶段
  3. afterRender() – 组件插入DOM后，与生成的DOM交互。
  4. render event – 在组件的render event去执行自定义渲染逻辑，会比自定义render()，afterRender()函数好很多。

在Renderer.js中重写默认的渲染函数

Render

组件初始化后就会执行,Lightning框架在init事件后会调用render()渲染函数.
render()函数通常返回一个DOM节点、一个DOM节点数组，或者什么也不返回。基本HTML组件在渲染组件时需要DOM节点。我们可以使用render函数来修改DOM或者修改组件的markup。

在render()函数中，在组件被渲染时调用helper方法修改DOM。

AfterRender

afterRender函数在render()函数之后被调用。AfterRender函数能够让我们与DOM树进行交互，修改已经完成渲染的组件，这个阶段组件实际上已经插入DOM Tree了。

Note :

  • Render和AfterRender函数只被调用一次，可以在函数中保留只需要运行一次的业务逻辑。
  • 函数只能被用来修改 DOM或者页面标记标签.
  • 不要在Render函数中执行DML操作。可以在AfterRender中去做。

Rerender

当组件因为用户操作(如按钮点击事件)而发生变化时，会调用Rerender函数。该函数会在组件重新渲染时更新组件。

Unrender

组件销毁时，会触发unrender事件。内置的unrender()函数会删除所有DOM节点，我们可以重写unrender()函数。
